# Skills Content Design Principles

## What is a Skill?

A skill is a directory under `actions/skills-mcp/skills/` that contains:
- `SKILL.md` — frontmatter metadata + body with routing guidance
- `references/` — one or more reference files (detailed technical content)

## SKILL.md Schema

```yaml
---
name: skill-name-kebab-case   # REQUIRED — must match folder name exactly
description: >                # REQUIRED — routing-optimized, multi-line ok
  Trigger phrases for when an agent should use this skill.
  Use when [scenario]. Do not use when [anti-pattern].
metadata:
  author: adobe-enterprise-architecture
  version: "1.0"
---
```

The `description` field is what agents see in `list_skills` output. Write it to be routing-optimized:
- Include trigger phrases an agent might search for
- State clearly when to use AND when NOT to use
- Be specific about the skill's scope

## SKILL.md Body

The body (after frontmatter) should contain:
1. **Role** — what this skill domain covers
2. **When to Load References** — conditional table: for each reference file, describe the scenario that should trigger loading it. This is the progressive disclosure mechanism.
3. **Core Concepts** — essential knowledge that fits in the skill body (not so large it needs its own reference file)
4. **Quick Reference table** — task → which reference file to load

Example:
```markdown
## When to Load References

- **ACTION_STRUCTURE.md** — Load when writing or reviewing any new action
- **ERROR_HANDLING.md** — Load when adding error responses or 4xx/5xx mapping
```

## Reference File Conventions

- One concern per file
- File names are SCREAMING_SNAKE_CASE.md (e.g. `ACTION_STRUCTURE.md`)
- Content is real and usable — copy-paste ready patterns, not vague summaries
- Include code examples where appropriate
- Keep files focused; avoid files >200 lines

## registry.json Schema

Generated by `scripts/generate-registry.js`. Do not edit manually.

```json
{
    "deployId": "skills-mcp-v1",
    "skills": [
        {
            "name": "skill-name",
            "description": "routing description...",
            "folder": "skill-name",
            "body": "# Skill body content...",
            "references": [
                {
                    "path": "references/FILENAME.md",
                    "filename": "FILENAME.md"
                }
            ]
        }
    ]
}
```

## Adding a New Skill

1. Create `actions/skills-mcp/skills/<skill-name>/`
2. Create `SKILL.md` with required frontmatter (`name` must match folder name)
3. Create `references/` with reference files
4. Run `node scripts/generate-registry.js` to verify no errors
5. Run `npm test` to verify the server still passes
6. Update `docs/design-principles/vision.md` skill domains table

## Validation

`scripts/generate-registry.js` validates:
- Skills directory is not empty
- Each skill folder has a `SKILL.md`
- `name` and `description` frontmatter fields are present
- Exits with code 1 on any validation error
